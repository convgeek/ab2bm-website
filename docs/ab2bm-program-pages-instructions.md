# AB2BM Program Pages — Web Copy & Component Instructions

This document provides content and implementation instructions for updating the five ADVANCE program pages on the AB2BM website (`ab2bm-website.vercel.app`). There are three additions per program page:

1. **Program Icon** — A simple SVG outline icon for each program's hero/header area
2. **"How the Program Works" section** — A new stepped-progression section inserted *above* the existing "Optional Program Ingredients" section
3. **"What You Get" section** — A new card-based section inserted *below* "How the Program Works" and *above* "Optional Program Ingredients"

---

## Reference Design Pattern

- **"How the Program Works" progression:** Display steps left-to-right on desktop, top-to-bottom on mobile. Each step gets a numbered circle icon (1, 2, 3…), a short step name in bold, and body text below. Connect steps with a subtle arrow or line between them to convey progression. Use the site's existing design system (dark backgrounds, white/light text, clean spacing).
- **"What You Get" cards:** Follow the layout pattern from Conversational Geek's "Where We Focus" section at `conversationalgeek.com/services/content-creation` — a grid of cards, each with a small icon, a bold title, and a 1–2 sentence description. Likely a 2×2 or 3-column grid depending on item count.

## Implementation Notes

### Icon Generation
- All program icons should be rendered as inline SVGs or standalone `.svg` files
- Use a consistent stroke width (1.5–2px), consistent viewBox (e.g., `0 0 24 24` or `0 0 48 48`), and `currentColor` for the stroke so they inherit the page's text color
- Icons should work at both small sizes (24×24 in nav or cards) and larger sizes (48×48 or 64×64 in hero areas)
- Style should match a clean outline icon set like Lucide or Feather icons — not filled, not overly detailed

### Section Ordering (per program page)
The final page structure for each program should be:
1. Hero (program name, subtitle, Solution Overview, Best For)
2. **How the Program Works** ← NEW
3. **What You Get** ← NEW
4. Optional Program Ingredients (existing)
5. Compliance-First by Design (existing)
6. Full-Service Program Management (existing)
7. CTA (existing)

### "How the Program Works" Component
- Desktop: horizontal left-to-right stepped progression with connector lines/arrows between steps
- Mobile: vertical top-to-bottom stack
- Each step: numbered circle (1, 2, 3…), bold step name, body text
- Body text for every step starts with an active verb

### "What You Get" Component
- Card grid layout matching the "Where We Focus" pattern on `conversationalgeek.com/services/content-creation`
- Each card: small icon (from the icon descriptions in the tables above), bold title, 1–2 sentence description
- Desktop: 2×2 grid for programs with 4 items
- Cards should have subtle borders or background differentiation, consistent spacing
