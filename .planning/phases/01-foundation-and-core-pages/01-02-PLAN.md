    **Section component requirements:**

    **Hero (`src/components/sections/Hero.tsx`):**
    Props: `{ headline: string; subheadline: string; ctaLabel: string; ctaHref: string }`.
    Layout: Full-width section, centered content, large headline, subheadline, primary CTA button (shadcn/ui Button).     **LogoStrip (`src/components/sections/LogoStrip.tsx`):**
    Props: `{ logos: Array<{ url: string; alt: string }> }`.
    Layout: Horizontal scrolling row of logos using Next.js `<Image>`. If logos array is empty, render nothing (section hidden). In Phase 1 this will be empty until client provides logos — that is expected.

    **ProgramsOverview (`src/components/sections/ProgramsOverview.tsx`):**
    Props: `{ programs: Array<{ _id: string; name: string; slug: { current: string }; tagline: string; ctaLabel: string }> }`.
    Layout: 2-column card grid (1 col on mobile). Each card shows program name, tagline, and a CTA button linking to `/programs#${slug.current}`. If programs array is empty, render a placeholder with hardcoded program names so the section is never blank at Phase 1.

    **AudienceStats (`src/components/sections/AudienceStats.tsx`):**
    Props: `{ stats: Array<{ label: string; value: string; footnote?: string }> }`.
    Layout: Horizontal stat bar or 4-column grid. If stats array is empty, use hardcoded placeholder stats: `[{ label: "Total Audience", value: "50,000+", footnote: "Verified opt-in" }, { label: "MSPs", value: "12,000+" }, { label: "MSSPs", value: "4,000+" }, { label: "IT Decision-Makers", value: "18,000+" }]`. Placeholder stats are labeled "(estimate)" in the footnote to be honest with visitors until real numbers are available from Phase 2.

    **TestimonialHighlight (`src/components/sections/TestimonialHighlight.tsx`):**
    Props: `{ quote: string; attribution: string; role: string; company: string } | null`.
    Layout: Blockquote styled card. If null (no testimonial in Sanity), render a placeholder: a muted quote block with "Client testimonials coming soon." — never a crash or empty section.

    **BlogPreview (`src/components/sections/BlogPreview.tsx`):**
    Props: `{ posts?: Array<{ _id: string; title: string; slug: string; excerpt: string; publishedAt: string }> }`.
    Layout: 3-column card grid. Per RESEARCH.md Pitfall 6: if `posts` is empty or undefined, render NOTHING (the section is absent from the page). This is intentional — blog is Phase 2. Do NOT render "Coming soon" or an empty grid.

    **FooterCta (`src/components/sections/FooterCta.tsx`):**
    Props: `{ headline: string; body: string; ctaLabel: string }`.
    Layout: Full-width dark background section. Two-path CTAs: primary button linking to `/contact` (inquiry), secondary link to `/contact?type=media-kit` (media kit download). Use shadcn/ui Button variants.

    **Styling constraint:** All components use Tailwind v4 utility classes only — no custom CSS. Use shadcn/ui Button and Card primitives where appropriate. Responsive: mobile-first, breakpoints at `md:` and `lg:`.

    **stegaClean() reminder:** Import `stegaClean` from `@sanity/client/stega` and wrap any Sanity string used in an `href`, conditional, or className before comparison. Document with comment: `// stegaClean() — prevents Visual Editing encoding from corrupting string comparisons`.
