    **`src/lib/actions/submit-inquiry.ts`:**

    Zod schema:
    ```typescript
    const InquirySchema = z.object({
      firstName: z.string().min(1, 'First name is required'),
      lastName: z.string().min(1, 'Last name is required'),
      email: z.string().email('Valid email required'),
      company: z.string().min(1, 'Company is required'),
      programType: z.enum(['content-syndication', 'webinar', 'general']),
      message: z.string().optional(),
      hutk: z.string().optional(), // HubSpot tracking cookie — sent from client
    })
    ```

    Server action flow:
    1. `'use server'` directive at top of file
    2. Zod `safeParse` — return `{ error: parsed.error.flatten().fieldErrors }` on failure
    3. Supabase insert wrapped in `try/catch` — log error, DO NOT re-throw (Supabase failure must not block HubSpot)
    4. HubSpot Forms API POST — include `hutk` in `context` if present (see RESEARCH.md Pitfall 1)
    5. HubSpot CRM API POST — create deal with `program_type__c` property (requires HUBSPOT_PRIVATE_APP_TOKEN in Authorization header)
    6. Return `{ success: true }` or `{ error: string }` from HubSpot failures

    HubSpot auth separation (RESEARCH.md Pitfall 7):
    - Forms API (`api.hsforms.com`): NO Authorization header — uses portalId + formGuid in URL only
    - CRM Deals API (`api.hubspot.com`): `Authorization: Bearer ${process.env.HUBSPOT_PRIVATE_APP_TOKEN}`

    If `process.env.HUBSPOT_PORTAL_ID` is undefined (local dev without .env.local), log a warning and skip HubSpot calls — return `{ success: true, dev: true }` to avoid blocking development. Add a `DEV_MOCK_HUBSPOT=true` env var check.
