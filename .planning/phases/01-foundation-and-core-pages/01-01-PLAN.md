    **Step 1 — Scaffold:**
    Run `npx create-next-app@latest . --typescript --tailwind --app --src-dir --import-alias "@/*"` from the project root. Accept all defaults. This initializes the App Router structure with TypeScript and Tailwind v4.

    **Step 2 — Install dependencies:**
    ```bash
    npm install next-sanity @sanity/client @sanity/image-url groq @portabletext/react
    npm install react-hook-form @hookform/resolvers zod
    npm install @supabase/supabase-js @supabase/ssr
    npm install next-sitemap
    npm install -D @playwright/test vitest @vitejs/plugin-react prettier prettier-plugin-tailwindcss
    npx playwright install --with-deps chromium
    npx shadcn@latest init
    ```
    If `npx shadcn@latest init` writes a `tailwind.config.ts`, delete it — Tailwind v4 uses CSS-first config in `globals.css` only. Verify CSS custom properties are in `:root` in `globals.css`.

    **Step 3 — Sanity project:**
    Run `npx sanity@latest init` inside the project directory to create a new Sanity project. Choose a new project, name it "Advance B2B Media", use the "Clean project" template, dataset name `production`. Note the projectId. Add to `.env.local`:
    ```
    NEXT_PUBLIC_SANITY_PROJECT_ID=<from init>
    NEXT_PUBLIC_SANITY_DATASET=production
    SANITY_API_READ_TOKEN=<create in sanity.io/manage — Viewer token>
    ```

    **Step 4 — `src/sanity/lib/client.ts`:**
    ```typescript
    import { createClient } from 'next-sanity'

    export const client = createClient({
      projectId: process.env.NEXT_PUBLIC_SANITY_PROJECT_ID!,
      dataset: process.env.NEXT_PUBLIC_SANITY_DATASET!,
      apiVersion: '2026-02-01',
      useCdn: true,
    })
    ```

    **Step 5 — `src/sanity/lib/live.ts`:**
    ```typescript
    import { defineLive } from 'next-sanity'
    import { client } from './client'

    export const { sanityFetch, SanityLive } = defineLive({
      client: client.withConfig({ apiVersion: '2026-02-01' }),
      serverToken: process.env.SANITY_API_READ_TOKEN,
      browserToken: process.env.SANITY_API_READ_TOKEN,
    })
    ```

    **Step 6 — Draft mode routes:**
    Create `src/app/api/draft-mode/enable/route.ts`:
    ```typescript
    import { draftMode } from 'next/headers'
    import { redirect } from 'next/navigation'
    import { validatePreviewUrl } from '@sanity/preview-url-secret'
    import { client } from '@/sanity/lib/client'

    export async function GET(request: Request) {
      const { isValid, redirectTo = '/' } = await validatePreviewUrl(client, request.url)
      if (!isValid) return new Response('Invalid secret', { status: 401 })
      ;(await draftMode()).enable()
      redirect(redirectTo)
    }
    ```
    Create `src/app/api/draft-mode/disable/route.ts` that calls `(await draftMode()).disable()` and redirects to `/`.

    **Step 7 — Sanity Studio route:**
    `src/app/studio/[[...tool]]/page.tsx`:

    ```typescript
    export const dynamic = 'force-dynamic'
    export default function StudioPage() {
      return <NextStudio config={config} />
    }
    ```

    **Step 8 — Root layout `src/app/layout.tsx`:**
    Import and render `SanityLive`, `VisualEditing` (when Draft Mode active), and `GoogleTagManager`. Structure:
    ```tsx
    import { SanityLive } from '@/sanity/lib/live'
    import { VisualEditing } from 'next-sanity/visual-editing'
    import { draftMode } from 'next/headers'
    import { GoogleTagManager } from '@next/third-parties/google'

    export default async function RootLayout({ children }: { children: React.ReactNode }) {
      return (
        <html lang="en">
          <body>
            <GoogleTagManager gtmId={process.env.NEXT_PUBLIC_GTM_ID!} />
            {children}
            <SanityLive />
            {(await draftMode()).isEnabled && <VisualEditing />}
          </body>
        </html>
      )
    }
    ```
    Add `NEXT_PUBLIC_GTM_ID=GTM-XXXXXXX` to `.env.local.example` (real GTM ID provided by client — use placeholder in dev).

    Define all Sanity document types that Pages 01-02 through 01-04 will consume. These are the contracts — downstream plans import GROQ queries and rely on these field names.

    **`src/sanity/schemas/documents/homepage.ts`** — Singleton (`__experimental_actions: ['update', 'publish']`):
    Fields: `heroHeadline` (string), `heroSubheadline` (text), `heroCta` (string), `heroCtaHref` (string, default `/contact`), `clientLogos` (array of images with `alt` field), `programsOverview` (array of references to `program`), `audienceStats` (array of objects: `{label: string, value: string, footnote: string}`), `testimonialHighlight` (reference to `testimonial`), `blogPreviewHeadline` (string), `footerCtaHeadline` (string), `footerCtaBody` (text), `footerCtaLabel` (string).

    **`src/sanity/schemas/documents/program.ts`**:
    Fields: `name` (string, required), `slug` (slug, source: name), `tagline` (string), `programType` (string, enum: `content-syndication` | `webinar`), `order` (number, for ordering on page), `mechanics` (Portable Text — rich text block), `deliverables` (Portable Text), `targetingOptions` (Portable Text), `ctaLabel` (string), `ctaHref` (string).

    **`src/sanity/schemas/documents/teamMember.ts`**:
    Fields: `name` (string), `role` (string), `bio` (text), `photo` (image with hotspot), `order` (number).

    **`src/sanity/schemas/documents/testimonial.ts`**:
    Fields: `quote` (text), `attribution` (string — full name), `role` (string), `company` (string), `logo` (image, optional).

    **`src/sanity/schemas/documents/siteSettings.ts`** — Singleton:
    Fields: `siteTitle` (string), `siteDescription` (text), `twitterHandle` (string), `defaultOgImage` (image).

    **`src/sanity/sanity.config.ts`**:
    Import and register all document types in the `schema.types` array. Use `defineConfig`. Set `projectId` and `dataset` from env vars.

    **`src/sanity/lib/queries.ts`** — Define typed GROQ queries using `defineQuery`:
    - `HOMEPAGE_QUERY` — fetches all homepage document fields including referenced programs (by `_id`, `name`, `slug`, `tagline`, `ctaLabel`) and testimonialHighlight
    - `PROGRAMS_QUERY` — `*[_type == "program"] | order(order asc)` with all fields
    - `TEAM_MEMBERS_QUERY` — `*[_type == "teamMember"] | order(order asc)`
    - `TESTIMONIALS_QUERY` — all testimonials
    Export each query constant. See RESEARCH.md Code Examples section for exact GROQ patterns.

    **`src/types/sanity.ts`**:
    Run `npx sanity@latest typegen generate` to auto-generate TypeScript types from the schema. If typegen isn't available in this version, manually create TypeScript interfaces matching each document type (must match field names exactly).

    **Critical constraints:**
    - `program.programType` MUST use the exact enum values `content-syndication` and `webinar` — these values are referenced by the URL param pattern in CONV-05 and the server action in Plan 01-05
    - All singleton schemas MUST have `__experimental_actions: ['update', 'publish']` to prevent duplicate document creation
    - Use `stegaClean()` in any component that uses a Sanity string in a conditional or class name — document this in code comments
